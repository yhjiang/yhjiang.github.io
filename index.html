Effective Objective-C 2.0第三章 (接口与API设计)

第15条: 用前缀避免命名空间冲突

Objective-C没有其他语言那种内置的命名空间（namespace）机制，所以起名的时候要设法避免潜在的命名冲突，要不然程序中会出现”重名符号错误”(duplicate symbol error)。解决方法：为所有的名称都加上适当前缀。 Apple宣称其保留使用所有“两字母前缀”(two-letter prefix)的权利，所以自己选用的前缀应该是三个字母的。举个栗子，开发者使用TW字母作为前缀，可能会跟系统自带的Twitter框架产生冲突。

要点

  • 选择与你的公司，应用程序或者两者都有关联的名称作为类名的前缀，并且在所有的代码中均使用这一前缀。(类名，分类，方法，都应加上相关前缀)
  • 如果是自己开发的程序库中用到了第三方库，应该为第三方库中的名称加上前缀防止冲突。

第16条: 提供”全能初始化方法”

要点

  • 在类中提供一个全能的初始化方法，并在文档中指明，其他的初始化方法都应该调用此方法。
  • 如果全能初始化方法与父类的不同，则需要在子类重写父类中对应方法。

第17条: 实现description方法

实现自定义类的description方法

代码

- (NSString *)description { return [NSString stringWithFormat:@"<%@: %p, \n %@ \n%@>",[self class], self, _user_nickname,_user_avatar]; }

在新实现的description方法中，应该也像默认的实现那样，打印出类的名字和指针地址，因为这些内容也许会用到。

如果自定义类中带有的属性较多，可以把待打印的信息放到字典中，然后将字典对象打印输出。

示例

- (NSString *)description { return [NSString stringWithFormat:@"<%@: %p, \n %@>",[self class], self, @{@"nickname":_user_nickname, @"avtar":_user_avatar}]; }

Xcode有一款插件可以自动把属性转成description中打印，插件名为 xCodeGenerateDescriptionPlugin, 安装方法可通过Alcatraz安装或者跳转到该项目github主页把项目下载下来，xcode中build一下。

使用方法

  1. In the .h file of your class, select all the properties you want included in your - (NSString *)description method.
  2. In the edit menu select Make Description
  3. Plugin will do all the work for you.

要点

  • 实现description方法返回一个有意义的字符串，用于描述该实例。
  • 若想在调试中打印出更详尽的对象描述信息，则应实现debugDescription方法。
