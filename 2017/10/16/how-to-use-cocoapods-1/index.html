CocoaPods由浅入深---安装与使用（一）

Posted by Yinghui Jiang on 2017-10-16

介绍

一、 CocoaPods是什么？

CocoaPods是Swift和Objective-C的第三方类库管理工具。 它拥有超过38,000个类库并且有超过270万个app都在使用它。Cocopods可以帮你你优化你的项目。 — 来自官网介绍

二、CocoaPods好处

CocoaPods可以把项目中用到的第三方类库统一来管理，这大大减轻了我们手工地一个个导入第三方库，并且添加framwork，设置编辑参数等工作。在CocoaPods中,一句 pod install 搞定。

安装与使用

安装

一、升级Mac的ruby版本

Mac都自带ruby，用命令行直接输入以下命令
$ sudo gem update --system

弹出 Password: 直接输入本机的登录密码，点击回车

我电脑当前的的ruby版本

$ gem --version 2.6.14

二、替换ruby的源地址

因为ruby用的软件源 https://rubygems.org 使用的亚马逊的云服务被墙了，所以需要替换成国内的源， 原来的 https://ruby.taobao.org/ 现在已经停止维护了，现在统一交给ruby-china来维护了

当出现

表明已经成功替换成国内的源地址。

三、安装CocoaPods

终端输入

$ sudo gem install cocoapods

如果出现

说明没有权限，需要指定安装路径

安装成功之后的显示如下

成功安装了CocoaPods之后，下一步就是设置CocoaPods

第一次安装，这个过程有点漫长，快的话十几分钟，慢的话，有可能一两个小时。你会发现终端一直卡在 Setting up CocoaPods master repo 不动了，说明CocoaPods正在把它的信息下载到 ~/.cocoapods目录里。

你可以通过 command + n 新建一个终端窗口，执行 cd ~/.cocoapods/ 进入到该文件夹下，然后执行 du -sh * 来查看文件的大小，截止到目前，我的这个目录是1.2G左右。

当命令行中出现了 Setup completed 说明已经完成了。

下面就真正的进入了如何使用CocoaPods的步骤。

四、CocoaPods的使用

（1） 首先测试一下pod的搜索功能

终端输入 pod search AFNetworking ,
出现以下内容，说明CocoaPods功能已经可以正常使用。

（2）在工程中创建Podfile文件

首先进入你工程所在的目录，

创建Podfile文件（注意文件名字一定要正确）

这时候，你的工程目录下发现多了一个Podfile文件。

（3）编辑Podfile文件

终端中使用vim来编辑Podfile文件

进入下面的页面

按下键盘的 i 键，也就是insert的简称，进入编辑模式,这时候窗口底部显示 -- INSERT --字样，并且光标锁定在第一行，接下来输入以下文字

platform :ios, ‘8.0’ 表示的是，当前支持的iOS最低版本是8.0

target ‘(替换成你的工程名)’ do 中间替换成你对应的工程的名称

pod ‘AFNetworking’, ‘~> 3.1.0’ 指的是指定下载的版本为 v3.1.x,x大于等于0， 如果去掉后面的’~> 3.1.0’,就表示下载的AFNetWorking为最新版的。

编辑完成后，保存并关闭Podfile文件， 首先按下键盘左上角的 Esc按钮，其次输入 ： （先按住Shift键，再按：键），再输入wq，点击回车键，这时候就成功保存了刚才修改的内容。

（4）安装Podfile文件中的第三方库

保持终端执行命令所在的文件夹跟Podfile在同一个文件夹，不然就提示找不到Podfile文件， 执行

pod install

这时候，AFNetworing就成功的添加到工程中， 关闭XCode，再次打开工程目录会发现多了个后缀为xcworkspace的文件，以后打开工程的时候就双击打开这个文件就可以了（打开xcodeproj文件，编译会报错）

（5）工程中使用第三方库

在工程中需要用到AFNetworking中文件中，直接导入 #import <AFNetworking.h> , （注意要用’< >’来导入，而不是用 “ ”）

至此，CocoaPods的基本使用方法已经完了。

>
总结

  • CocoaPods的使用方法核心就一句话 ‘pod install’ ,在每次Podfile文件中新增三方库的时候，执行 ‘pod update’ 就可以。
